這一步需要你手動操作，因為它涉及到你的 Google 帳號。請依照以下步驟：

1.  前往 Google Cloud Console：

    - 點擊此連結：https://console.cloud.google.com/ (https://console.cloud.google.com/)
    - 如果你有多個專案，請在左上角選單選擇「新增專案」，並為它取一個名字（例如 My-Login-Project），然後建立。

2.  啟用 API：

    - 在左側導覽列，找到「API 和服務」 > 「已啟用的 API 和服務」。
    - 點擊上方的「+ 啟用 API 和服務」。
    - 搜尋「Google People API」並啟用它。這是為了讓我們能獲取使用者的名稱和頭像。

3.  設定 OAuth 同意畫面：

    - 在左側導覽列，找到「API 和服務」 > 「OAuth 同意畫面」。
    - 選擇「外部 (External)」，然後點擊「建立」。
    - 填寫必填欄位：
      - 應用程式名稱：給你的網站取個名字。
      - 使用者支援電子郵件：選擇你的 email。
      - 開發人員聯絡資訊：再次輸入你的 email。
    - 點擊「儲存並繼續」，接下來的「範圍」、「測試使用者」頁面都可以直接點擊「儲存並繼續」跳過。最後回到資訊
      主頁。

4.  建立憑證：

    - 在左側導覽列，找到「API 和服務」 > 「憑證」。
    - 點擊上方的「+ 建立憑證」，選擇「OAuth 用戶端 ID」。
    - 應用程式類型：選擇「網頁應用程式」。
    - 名稱：可以自訂，或用預設值。
    - 已授權的 JavaScript 來源：點擊「+ 新增 URI」，輸入你的開發環境網址，例如 http://localhost:80。
    - 已授權的重新導向 URI：點擊「+ 新增 URI」，輸入你的後端回呼 URL。根據我們的規劃，請填寫
      http://localhost:80/auth/google/callback。
    - 點擊「建立」。

5.  複製憑證：
    - 建立成功後，會彈出一個視窗，裡面有你的「用戶端 ID (Client ID)」和「用戶端密鑰 (Client Secret)」。
    - 請務必將這兩串字串複製下來，我們馬上就會用到。
