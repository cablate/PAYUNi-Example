## 🎯 學習心態宣言

### 這門課適合你，如果你...
- 想從「功能實現者」提升到「系統設計者」。
- 希望理解「為什麼」，而不只是照著做。
- 願意投入時間思考、練習、甚至失敗後重來。
- 想累積可轉移到其他專案的能力。
- 對真實世界的架構案例有興趣。

### 這門課可能不適合，如果你...
- 只想快速複製貼上程式碼就上線。
- 不想了解背後原理，只求「能動就好」。
- 沒時間完成練習或思考題。
- 只需要一個現成的金流工具（建議直接用 SaaS）。

> 可以嵌在任何導讀、FAQ、行銷頁面中，提醒學員評估期望值。

## 🤝 與 AI 共讀的 4 個步驟

1. **提問模式**（請 AI 說明結構或差異）
   ```
   「為什麼 01_SYSTEM_ARCHITECTURE 的 Webhook 要分 Return URL / Notify URL？」
   ```
2. **驗證理解**（請 AI 檢查自己的說法）
   ```
   「我認為 CSRF Token 的作用是 X，請幫我確認是否正確。」
   ```
3. **場景延伸**（讓 AI 協助移植到新專案）
   ```
   「如果要套用在訂閱制電商，哪些模組需要改？」
   ```
4. **除錯協助**（結合 TROUBLESHOOTING 指南）
   ```
   「npm start 時遇到 [錯誤]，我已經依照 TROUBLESHOOTING.md 做了 A/B，下一步？」
   ```

> 上傳文件給 AI 時，先傳 `00_DESIGN_PHILOSOPHY.md` 建立脈絡，再附相關章節或程式碼。
