## 🛑 為什麼你需要這個實戰包？

## 作為開發者

**串金流不只是把 API 接通而已。**

1. 想接通 API，要研究金流平台的文件 @@
2. 接通 API 後，還要開始構思前後端架構…
3. Webhook、Return url 是甚麼東西啊？他們的職責是甚麼？
4. 如何將金流方的狀態確實且安全地同步給自己的資料庫？
5. 資安問題呢？？
6. 上述做完，還有會員模組、訂單模組、商品模組、前端畫面、管理員後台… etc，
最後剩下多少時間可以專心處理屬於自己平台的這些業務邏輯開發呢？

而我的目的就是**降低串金流的阻礙與心理門檻，不需要從零研究、從零設計。**
**只需要理解、複製、貼上、然後準時下班！！！
多餘的時間好好應對業務邏輯與程式碼品質，而不是一直把時間心力花費在學習如何從頭設計流程。

同時，這也是一份不錯的作品集！你也將確切地具備「金流串接」「流程設計」的經驗。**
