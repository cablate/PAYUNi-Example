## 🛠️ 任務指引

### 任務 1：新增商品 (15 分)
檔案：`data/products.js`

這是一個靜態檔案 (模擬資料庫)。
1. 複製最後一個 `{}` 物件。
2. 改 `id` (不要重複)。
3. 改 `name` 和 `price`。
4. **重啟伺服器** (`npm start`)。

> **💡 為什麼要重啟？**
> 因為 `products.js` 是在伺服器啟動時載入記憶體的。這告訴你：如果是靜態資料，改了要重啟；如果是資料庫，改了不用重啟。

### 任務 2：換皮 (15 分)
檔案：`public/style.css`

我們用了 CSS 變數，所以改一個地方，全站都會變。
```css
:root {
  --primary-color: #FF6B6B; /* 改這裡 */
}
```
改完後，**強制重新整理** (Ctrl+Shift+R) 瀏覽器。

> **💰 商業價值筆記 (For Freelancers)：**
> 光是這一招「換皮」，你就能幫客戶做「品牌客製化」。
> 告訴客戶：「我不只幫你接金流，我還幫你把支付頁面設計成跟官網一樣的風格，增加信任感。」這就是你提高報價的理由。

### 任務 3：改文案 (10 分)
檔案：`index.html`

直接搜尋 `<h1>` 改標題。
這是最簡單的一步，但也是讓專案「變現」的第一步。

### 任務 4：全端擴充挑戰 (20 分)
這一步是驗證你是否真的懂了「資料流」。

1. **資料層**：`products.js` 加 `description: "..."`。
2. **前端層**：`app.js` 裡渲染 HTML 時加上 `<p>${product.description}</p>`。
3. **後端層**：`index.js` 的 `/api/create-order` 裡，把 `description` 塞進 payload。
4. **儲存層**：Google Apps Script (`code.gs`) 的 `doPost` 裡，把 `e.parameter.description` 寫進 Sheet。

> **💰 商業價值筆記 (For Freelancers)：**
> 客戶常會說：「我想要在訂單裡多記一個『備註』或『推薦人』。」
> 學會這一招，你就能回答：「沒問題，我可以客製化欄位。」而不是說：「這個套件不支援。」
